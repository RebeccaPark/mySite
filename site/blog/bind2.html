Reimplementing Function.prototype.bind for science

If you work with Javascript at all, sooner or later you will come across bind. It returns a function that uses the this keyword that you tell it to use. Another common use of bind is pre-populating function argument(s) for later use. For most of the time I have regarded bind as something that happens magically. But I decided to break the mystery and get to the nitty gritty of it by implementing my own versions. In this post I demonstrate my implementations of bind in ES5(bind2) and in ES6(bind3).

bind in detail

bind is a built-in Javascript method part of the prototype of the Function object. The first argument bind receives becomes the this keyword for the function returned by the bind method. Any following argument after the first is remembered to be used as arguments for the returned function.

Let's look at an example that uses bind. Later I use this example to demonstrate that my implementations work correctly.

Using bind

This example creates four buttons with callbacks to be executed on 'click' event. The callbacks are the methods for myDog object, an instance of the Dog prototype.

The Dog prototype is a representation of a dog. It has a property name that receives its value when an instance is initialized. It also has two methods sayName and sayWeight used as callbacks for the buttons. They respectively display the name and the weight of the dog in a DOM elmeent with identifier #result.

As an instance of Dog, I created myDog object to represent my lovely fur friend named Mumu. The buttons' callbacks are meant ot use information from myDog object. But because the first two buttons' callbacks are not bound to myDog, they lose the intended this keyword and fail to deliver the proper messages. Thankfully the last two buttons' callbacks are properly bound to the myDog object and display the message properly with the name 'Mumu'.

Now let's move onto the reimplementations!

bind2 with ES5


Function.prototype.bind2 = function() {
  var originalFunc = this;
  var self = arguments[0];
  var argsOuter = Array.prototype.slice.call(arguments, 1);
  
  var returnedFunc = function() {
    var argsInner = argsOuter.concat(Array.prototype.slice.call(arguments, 0));
    return originalFunc.apply(self, argsInner)
  }
  
  return returnedFunc;
}

bind2 extends the prototype of the Function object taking it after the original bind.

There are three objects that bind2 needs to pass onto returnedFunc for later use:
1. The function that bind2 is called on. We store that function in originalFunc.
2. The first argument passed on to the bind function. This is the object that needs to be the this. We store that object in self.
3. The arguments passed onto the bind function except for the very first argument. We store them in an array in argsOuter.
(*Note that to define argsOuter, we extrapolate slice method from the Array prototype. This is because although arguments is an Array-like object, it is not a real Array and does not have the Array methods. You can read more about it from MDN)

When returnedFunc is invoked, it returns the result of invoking originalFunc with apply method. The apply method takes the self object (number 2 from above) as its first argument to reference for the this object. The second argument argsInner is an array of arguments that passes onto originalFunc to be used as its own arguments. Note that argsInner is argsOuter concatenated with the arguments object of returnedFunc. This makes sure that any additional argument passed while invoking returnedFunc is not lost.

Now on to ES6!

bind3 with ES6

For bind3, I will skip the parts we already covered in bind2 and focus on the different parts.


Function.prototype.bind3 = function(self, ...argsOuter) {
  return (...argsInner) => this.apply(self, [...argsOuter, ...argsInner]);
}

bind3 receives one argument named self plus any number of arguments. The ...argsOuter expression is an example of spread syntax. This means that any number of arguments received after the first argument is stored in argsOuter as an array. Similarly, the returned function also receives any number of arguments and store them in array argsInner.

In the previous example, we had to store the original function that bind2 was called on in originalFunc. We don't need to do this for bind3 because we can use the arrow function, which carries over the outer function's this. This means this.apply is equivalent to the previous example's originalFunc.apply.

Lastly, we concatenate the argsOuter and argsInner using the spread syntax again. This will expand the two arrays and put elemenets from both arrays in a single array.
